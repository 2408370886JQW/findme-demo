# “偶遇 (Encounter)” 模块交互流程详解

本文档详细描述 FIND ME 产品中“偶遇”模块从筛选用户到发起互动的完整交互流程。设计旨在通过流畅的地图操作和沉浸式的用户卡片，营造轻松自然的社交氛围。

## 1. 初始状态：地图视图 (Map View)

用户点击底部 TAB 的“偶遇”图标进入该模块，随即进入全屏地图视图。该界面以地图为背景，直观展示当前定位附近的地理信息。地图上散布着附近用户的圆形头像（Avatar Pins），这些头像带有微弱的呼吸动效，生动地暗示了用户的活跃状态。顶部栏设有悬浮的搜索框与筛选按钮，采用半透明毛玻璃效果，既保证了功能的易用性，又保持了视觉上的通透感。

## 2. 步骤一：筛选兴趣标签 (Filter)

当用户希望寻找特定类型的社交对象（如“运动搭子”或“饭搭子”）时，可以点击顶部栏右侧的“筛选”图标或胶囊状的快捷标签栏。

**界面变化与交互反馈**

| 阶段 | 界面表现 | 交互细节 |
| :--- | :--- | :--- |
| **触发筛选** | 底部弹出一个半屏的筛选抽屉 (Bottom Sheet)。 | 抽屉内展示性别、年龄段及核心的“兴趣标签”（如 #健身、#探店）。标签采用胶囊样式，选中时变为品牌橙色 (#FF5500) 并伴有轻微震动反馈。 |
| **确认筛选** | 用户点击“确定”后，抽屉向下滑出消失。 | 地图视图发生**动态重绘**：不符合条件的用户头像淡出消失，符合条件的头像保留并重新聚类。地图中心会自动缩放，以最佳视野展示筛选结果。 |

## 3. 步骤二：浏览与选中用户 (Select User)

用户在筛选后的地图上通过拖动浏览，寻找感兴趣的人。当点击某个特定用户的头像时，界面会给予明确的反馈。

被点击的头像会放大 1.2 倍，并出现橙色光圈高亮，表示选中状态。此时，屏幕底部会弹出一个轻量级的“用户预览卡片” (Preview Card)。该卡片背景采用轻微的磨砂玻璃质感，不完全遮挡地图背景，内容包括用户昵称、年龄、距离（如 500m）以及 1-2 个核心兴趣标签。若用户点击地图空白处，预览卡片将自动收起，头像恢复默认状态。

## 4. 步骤三：查看详细资料 (View Profile)

若用户对预览卡片中的信息感兴趣，可以通过点击卡片或向上滑动卡片进入详情页。

预览卡片将通过**共享元素转场 (Shared Element Transition)** 平滑展开为全屏的“用户详情页” (Profile Page)。页面的视觉焦点位于顶部，展示用户的高清生活照或个人形象视频。向下滑动页面，用户可以查看详细的个人简介、发布的动态（关联“圈子”数据）以及常去的地点（关联“相见”数据）。底部常驻一个显眼的“打招呼”操作栏，方便用户随时发起互动。

## 5. 步骤四：发起互动 (Interact)

当用户决定与对方建立联系时，点击底部的“打招呼”按钮即可触发互动面板。

**互动方式与反馈**

| 互动类型 | 描述 |
| :--- | :--- |
| **发送表情** | 快速发送一个可爱的动态表情（如挥手、比心），适合轻量级打招呼。 |
| **AI 辅助破冰** | 系统根据双方的共同标签（如都喜欢“摄影”），自动生成 3 条开场白建议（例如：“看你也喜欢胶片摄影，平时去哪里扫街呀？”），降低社交压力。 |
| **直接私信** | 支持输入文字直接发送消息。 |

发送成功后，互动面板会自动收起，屏幕上方会出现“消息已发送”的 Toast 提示。系统随后会自动跳转或引导用户前往“消息 (Chat)”模块查看后续回复，完成社交闭环。

---

**总结**：整个流程从宏观的地图筛选，层层递进到微观的个人详情，最后通过低门槛的互动工具完成社交破冰。视觉上强调“透气感”和“流畅性”，操作上充分考虑了单手交互的便捷性。
