## 1. 产品概述 (Product Overview)

### 1.1 产品定位
"FIND ME" 是一款基于**场景化社交驱动**的生活方式发现应用。不同于传统点评类应用（如大众点评）的“搜索-评价”逻辑，FIND ME 侧重于**“场景-情绪-决策”**。它通过沉浸式的地图交互和精细化的场景分类（如“感情升温”、“闺蜜叙旧”），帮助用户在特定社交语境下快速找到最合适的吃喝玩乐去处。

### 1.2 核心价值
*   **场景化决策**：解决“不知道去哪”的痛点，通过具体场景（如“求婚策划”、“深夜微醺”）缩短决策路径。
*   **情绪价值导向**：不仅提供店铺信息，更强调氛围感、出片率等情绪价值指标。
*   **社交裂变**：通过高颜值的分享海报和动态优惠券，激发用户的分享欲，实现自然增长。

### 1.3 目标用户
*   **Z世代及年轻白领**：追求生活品质，注重仪式感和社交体验。
*   **情侣/密友**：需要高质量约会或聚会场所的人群。
*   **探店达人**：热衷于发现新奇特店铺并分享社交媒体的用户。

## 2. 核心功能逻辑 (Core Features & Logic)

### 2.1 首页与导航 (Home & Navigation)
**逻辑描述**：
首页采用“左侧场景导航 + 右侧地图/列表联动”的布局，模拟高德地图的沉浸式体验。

*   **侧边栏导航 (Sidebar)**：
    *   **一级分类**：情侣套餐、闺蜜套餐、兄弟套餐、情趣套餐等 8 大类。
    *   **二级场景**：点击一级分类展开（手风琴效果），显示具体场景（如“浪漫约会”、“纪念日”）。
    *   **交互**：点击二级场景，右侧列表和地图点位实时过滤刷新。
*   **地图模式 (Map Mode)**：
    *   默认展示列表，点击“地图”按钮切换至全屏地图模式。
    *   展示当前筛选下的所有商家 POI（兴趣点）。
    *   点击 POI 弹窗展示商家简要信息（名称、评分、人均、距离）。
*   **智能推荐 (Smart Recommendation)**：
    *   **入口**：位于顶部筛选栏左侧，带有呼吸动画的“猜你喜欢”徽章。
    *   **逻辑**：基于**当前时间段**（早/午/晚/深夜）和**用户位置**推荐 2-3 个高分或应景商家。
        *   *06:00-11:00*：推荐咖啡/早茶。
        *   *11:00-14:00*：推荐正餐/约会餐厅。
        *   *14:00-17:00*：推荐下午茶/甜点。
        *   *17:00-21:00*：推荐晚餐/特色菜。
        *   *21:00-06:00*：推荐酒吧/烧烤/夜宵。
    *   **展示**：横向滚动卡片，不干扰主列表浏览。

### 2.2 商家列表与详情 (Shop List & Details)
**逻辑描述**：
提供高效的筛选和丰富的信息展示，辅助用户决策。

*   **列表筛选**：
    *   支持多维度组合筛选：价格区间（<200, 200-500, >500）、距离范围（<1km, 1-3km, >3km）、好评优先、人均高低。
    *   **移动端适配**：筛选栏支持横向滑动，节省屏幕空间。
*   **商家卡片**：
    *   展示：封面图（左上角场景标签）、店名、评分（高亮）、人均价格、距离、特色标签（Top 3）。
    *   **团购透出**：如有优惠套餐，直接在卡片底部展示“团”字样及价格对比。
*   **详情交互**：
    *   点击卡片进入详情页（或弹窗）。
    *   **图片轮播**：支持左右滑动查看商家环境图/菜品图。
    *   **收藏功能**：点击爱心图标收藏/取消收藏，数据本地持久化存储。

### 2.3 社交分享与裂变 (Social Sharing)
**逻辑描述**：
将商家信息转化为高颜值的社交货币，并附带营销属性。

*   **海报生成**：
    *   点击详情页“分享”按钮，弹出海报预览。
    *   **内容构成**：品牌 Logo (FIND ME) + 商家主图 + 评分/人均 + 核心标签 + **动态优惠券** + 二维码。
*   **动态优惠券 (Dynamic Coupon)**：
    *   **样式**：拟物化票券设计，带虚线撕裂边框。
    *   **逻辑**：自动生成唯一核销码（如 `FINDME-888`），显示折扣力度（如“8.8折”）。
    *   **文案**：强调“限时优惠”、“截屏保存”，制造紧迫感。
*   **操作**：支持一键保存图片到本地相册。

### 2.4 路线导航 (Route Navigation)
**逻辑描述**：
闭环用户的到店体验，无需跳出 App。

*   **路径规划**：
    *   在地图模式或详情页点击“导航”。
    *   基于用户当前坐标（模拟）和商家坐标，绘制步行/驾车路线。
    *   显示预估距离和时间。

## 3. 数据结构 (Data Structures)

### 3.1 商家 (Shop)
```typescript
interface Shop {
  id: string;             // 唯一标识
  name: string;           // 店名
  rating: number;         // 评分 (0-5)
  price: number;          // 人均价格
  distance: string;       // 距离显示文本
  coordinates: {          // 地理坐标
    lat: number;
    lng: number;
  };
  tags: string[];         // 特色标签 (e.g., "法式", "露台")
  imageUrl: string;       // 封面图 URL
  description: string;    // 商家简介
  packageType: string;    // 一级分类 ID (e.g., "couple")
  sceneTheme: string;     // 二级场景 ID (e.g., "date")
  deals?: Deal[];         // 团购套餐列表
  reviews?: Review[];     // 用户评价列表
}
```

### 3.2 团购套餐 (Deal)
```typescript
interface Deal {
  title: string;          // 套餐标题
  price: number;          // 优惠价
  originalPrice: number;  // 原价
  tags: string[];         // 套餐标签 (e.g., "随时退")
}
```

### 3.3 分类 (Category)
```typescript
interface Category {
  id: string;             // 分类 ID
  name: string;           // 分类名称 (e.g., "情侣套餐")
  subTitle: string;       // 副标题 (e.g., "甜蜜升温")
  icon: IconComponent;    // 图标组件
  subCategories: SubCategory[]; // 子分类列表
}
```

## 4. 非功能性需求 (Non-Functional Requirements)

### 4.1 UI/UX 设计规范
*   **风格**：高德 (Amap) 风格，强调清晰的信息层级和地图交互。
*   **视觉效果**：
    *   **Glassmorphism**：导航栏、侧边栏、浮层使用背景模糊 (backdrop-blur) 效果，提升通透感。
    *   **阴影与圆角**：使用细腻的阴影 (box-shadow) 和大圆角 (rounded-xl) 营造现代感。
*   **排版**：
    *   一级菜单：14px，加粗，高对比度。
    *   二级菜单：11px，精致，选中时高亮。
    *   正文：12-14px，注重阅读舒适度。
*   **响应式**：优先适配移动端 (Mobile First)，同时兼容桌面端宽屏显示。

### 4.2 性能要求
*   **加载速度**：首屏加载时间 < 1.5s。
*   **交互响应**：点击反馈延迟 < 100ms。
*   **海报生成**：图片生成耗时 < 2s。

## 5. 未来规划 (Roadmap)

### Phase 2: 用户体系与个性化 (User System)
*   **个人中心**：注册/登录，管理收藏夹、浏览历史、发布的评价。
*   **评价系统**：支持用户上传图文评价，打分，形成社区氛围。

### Phase 3: 智能化升级 (Intelligence)
*   **天气联动**：接入天气 API，雨天推荐室内活动，晴天推荐露台/户外。
*   **千人千面**：基于用户历史行为（点击、收藏）优化“猜你喜欢”算法。

### Phase 4: 商业化闭环 (Monetization)
*   **在线核销**：支持 App 内直接购买团购套餐并生成核销码。
*   **商家入驻**：提供商家端后台，管理店铺信息和发布优惠。
