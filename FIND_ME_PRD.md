## 1. 产品概述 (Product Overview)

### 1.1 产品背景
在当前的本地生活服务市场中，用户对于“去哪儿玩”、“吃什么”的需求已经从单纯的信息搜索转变为场景化的决策。传统的列表式展示缺乏代入感，无法满足用户在特定社交场景（如情侣约会、闺蜜聚会、兄弟聚餐）下的情感诉求。

### 1.2 产品定位
**FIND ME** 是一款专注于**场景化社交消费决策**的团购展示平台。它通过精致的视觉设计和沉浸式的交互体验，帮助用户在不同的社交关系（情侣、闺蜜、兄弟、情趣等）下快速找到最合适的消费场所和套餐。

### 1.3 核心价值
*   **场景化决策**: 摒弃传统的品类分类（如火锅、西餐），采用社交关系分类（如两人约会、闺蜜聚会），直击用户核心诉求。
*   **精致交互体验**: 采用高德“扫街榜”式的精致侧边栏设计，结合胶囊式动效和透气感排版，提供高品质的浏览体验。
*   **智能推荐**: 基于时间段和场景的智能推荐算法，为用户提供“猜你喜欢”的个性化服务。

## 2. 用户角色 (User Personas)

| 角色 | 描述 | 核心需求 |
| :--- | :--- | :--- |
| **场景探索者** | 这里的用户通常没有明确的目的地，但有明确的社交对象（如约会对象、朋友）。 | 希望根据社交关系快速筛选出氛围合适、品质有保障的店铺。 |
| **品质追求者** | 对环境、服务和视觉体验有较高要求的用户。 | 偏好精致的UI设计，通过高质量的图片和排版来判断店铺的调性。 |
| **效率型用户** | 希望快速决策，直接查看套餐详情和价格的用户。 | 需要清晰的价格筛选、距离筛选和直观的套餐内容展示。 |

## 3. 功能需求 (Functional Requirements)

### 3.1 核心导航系统 (Core Navigation)

#### 3.1.1 侧边栏导航 (Sidebar Navigation)
*   **设计风格**: 采用高德“扫街榜”风格，追求精致、轻量化。
*   **一级菜单**:
    *   **分类**: 情侣套餐、闺蜜套餐、兄弟套餐、情趣套餐。
    *   **交互**: 点击一级菜单，展开对应的二级子菜单；选中状态下文字高亮（品牌橙色 #FF5500），非选中状态为深灰色 (#333333)。
    *   **视觉**: 去除冗余装饰（如麦穗），保持清爽；字体字号适中，强调透气感。
*   **二级菜单**:
    *   **展示形式**: 展开在一级菜单下方。
    *   **首项样式**: 第一个子选项采用**胶囊样式**（背景色 #FF5500，白字），突出推荐感。
    *   **其他项样式**: 后续子选项保持纯文本展示，选中时高亮，未选中时为标准文本，层次分明。
    *   **交互**: 点击任意二级选项，右侧内容区域实时刷新对应分类的店铺列表。
*   **移动端适配**:
    *   在移动设备上，侧边栏默认收起。
    *   顶部导航栏左侧提供“菜单”按钮（汉堡图标），点击后从左侧滑出抽屉式导航（Drawer），内容与桌面端侧边栏一致。

#### 3.1.2 顶部工具栏 (Top Toolbar)
*   **搜索功能**: 支持关键词搜索（如“西餐”、“酒吧”），实时过滤店铺列表。
*   **快捷操作**:
    *   **暗色模式切换**: 支持日间/夜间模式一键切换。
    *   **我的订单**: 点击弹出订单列表浮层，查看历史订单状态。
    *   **收藏夹**: 点击查看已收藏的店铺列表。
    *   **地图模式**: 切换列表/地图视图，地图上显示店铺分布锚点。

### 3.2 内容展示系统 (Content Display)

#### 3.2.1 店铺列表 (Shop List)
*   **卡片设计**: 采用大图卡片设计，突出店铺环境和菜品。
*   **信息展示**:
    *   店铺名称、评分（支持小数，如 4.9）、人均价格。
    *   标签系统：显示“法式”、“露台”、“夜景”等特征标签。
    *   距离提示：显示距离用户当前位置的距离（如 500m）。
*   **套餐信息**:
    *   在店铺卡片下方直接展示核心套餐价格（原价 vs 团购价）。
    *   “抢”字图标突出优惠属性。

#### 3.2.2 筛选与排序 (Filter & Sort)
*   **筛选维度**:
    *   **价格**: 不限、低价 (<200)、中端 (200-500)、高端 (>500)。
    *   **距离**: 距离不限（特殊文案：“姬绮伟”）、近距离 (<1km)、中距离 (1-3km)、远距离 (>3km)。
    *   **排序**: 好评优先、人均高低。
*   **交互**: 点击筛选标签，列表实时更新；选中标签高亮显示。

### 3.3 详情与交互 (Details & Interaction)

#### 3.3.1 店铺详情 (Shop Details)
*   **点击交互**: 点击店铺卡片，右侧滑出/弹窗展示店铺详情。
*   **内容包含**:
    *   高清轮播图。
    *   详细地址与导航按钮。
    *   推荐菜品列表。
    *   用户评价摘要。
    *   “写评价”入口。

#### 3.3.2 订单与通知 (Orders & Notifications)
*   **通知系统**: 顶部右侧即时弹出通知（Toast），提示订单状态（如“您的订单即将过期”）。
*   **订单列表**: 展示订单详情，包括套餐名称、使用状态、有效期等。

## 4. 交互与UI规范 (Interaction & UI Guidelines)

### 4.1 色彩规范 (Color Palette)
*   **品牌主色**: `#FF5500` (活力橙) - 用于高亮、按钮、选中状态。
*   **辅助色**: `#FF8855` (浅橙) - 用于次级高亮、胶囊背景。
*   **背景色**:
    *   Light Mode: `#FFFFFF` (纯白背景), `#F5F5F5` (浅灰底色)。
    *   Dark Mode: `#1A1A1A` (深灰背景), `#000000` (纯黑底色)。
*   **文本色**:
    *   主要文本: `#333333` (深灰)。
    *   次要文本: `#666666` (中灰)。
    *   提示文本: `#999999` (浅灰)。

### 4.2 字体规范 (Typography)
*   **字体家族**: 优先使用系统默认无衬线字体 (San Francisco, PingFang SC, Roboto)。
*   **字号层级**:
    *   一级标题: 16px, Medium (适中字重，避免过粗)。
    *   二级标题: 14px, Regular (常规字重)。
    *   正文: 13px/12px, Regular。
    *   标签: 10px/11px, Medium。

### 4.3 动效规范 (Animation)
*   **侧边栏展开**: 使用 `transition-all duration-300` 实现平滑的高度变化。
*   **胶囊切换**: 选中状态切换时，背景色和文字颜色应有 200ms 的渐变过渡。
*   **列表加载**: 采用骨架屏 (Skeleton) 预加载，避免页面闪烁。

## 5. 技术要求 (Technical Requirements)

*   **前端框架**: React 19 + TypeScript。
*   **样式库**: Tailwind CSS 4 + shadcn/ui。
*   **图标库**: Lucide React。
*   **地图集成**: 集成高德地图/谷歌地图 API (通过组件封装)。
*   **响应式布局**: 必须兼容桌面端 (Desktop) 和移动端 (Mobile) 视图，断点设为 `md` (768px)。
